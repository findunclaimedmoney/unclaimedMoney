1. Project Overview
Build a PWA (Progressive Web App) that hosts a real-time, bidirectional voice-to-voice avatar. The core goal is an "empathic listener" experience.

2. Tech Stack Requirements
Frontend: React (Next.js) hosted on Vercel.

Orchestration: Node.js (Express) server to handle WebSockets.

Brain (LLM): OpenAI Realtime API (supports low-latency voice-to-voice).

Avatar Rendering: Hedra Streaming API (Client-side integration).

State/Memory: Supabase (PostgreSQL) for user authentication and chat history storage.

Audio Processing: Web Audio API (for real-time mic capture/playback).

3. Architecture A to Z
Step A: Environment Setup: Initialize a Next.js 15 project with TypeScript. Configure environment variables for OPENAI_API_KEY, HEDRA_API_KEY, and SUPABASE_URL.

Step B: The Bridge (WebSocket Layer): Create a secure WebSocket tunnel between the browser and the OpenAI Realtime API. This ensures the voice streaming is direct and low-latency.

Step C: The Avatar Integration: Integrate the Hedra SDK into the React component. Configure the onAudioChunk callback to feed audio output directly into the Hedra video renderer.

Step D: Empathy Layer (System Prompting): Define the system prompt: "You are Mia, an empathetic listener. Focus on active listening, short empathetic reflections, and avoiding long-winded answers. Prioritize user comfort."

Step E: Persistence: Implement a hook that saves summarized conversation snapshots to Supabase so that "Mia" can reference previous interactions.

Step F: Deployment: Set up a vercel.json configuration to handle high-concurrency WebSocket connections and enable PWA manifest for "app-like" behavior on mobile.

4. API Integrations to Enable
OpenAI Realtime API: wss://[api.openai.com/v1/realtime](https://api.openai.com/v1/realtime) (Required for sub-500ms response).

Hedra Streaming: Configure the streaming_url in the environment to point to their /stream endpoint.

ElevenLabs (Optional/Backup): If OpenAI's native voices are insufficient, integrate ElevenLabs as a secondary TTS provider via their WebSocket API.
